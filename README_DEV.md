# 🖼️ Image Extractor

> Extract images from files — run locally from source, no installation required.

This guide explains how to set up and run the **Image Extractor** application directly from its Python and Svelte source code, without compiling or using the standalone `.exe` file.

This is the perfect approach if you want to **modify the code** or run it locally in a **development environment**.

---

## 📋 Prerequisites

Before you begin, make sure the following are installed on your system:

| Requirement | Minimum Version | Download |
|-------------|----------------|----------|
| **Python** | 3.11+ | [python.org](https://www.python.org/downloads/) |
| **Node.js** | 20+ | [nodejs.org](https://nodejs.org/) |

---

## 🚀 Local Development Setup

Follow these three steps in order to get the application running.

### 1. Install Backend Dependencies

Open your terminal in the **root directory** of the project and install the required Python packages:

```bash
pip install -r requirements.txt
```

### 2. Install Frontend Dependencies & Build

Navigate to the `frontend` folder, install the required Node packages, and compile the Svelte user interface:

```bash
cd frontend
npm install
npm run build
cd ..
```

> This step uses **Vite** to bundle the Svelte UI into static files that the Python backend will serve.

### 3. Run the Application

Once the frontend is built, return to the root directory and start the application:

```bash
python main.py
```

The application should now be running. Open the URL shown in your terminal to access the UI.

## ✏️ Making Changes to the Code

Depending on what you change, you may or may not need to rebuild:

### Frontend Changes

If you modify the user interface — any `.svelte` or `.js` files inside `frontend/src/` — you must rebuild the frontend so Vite can package your updates:

```bash
cd frontend
npm run build
```

Then restart `python main.py`.

### Backend Changes

If you modify the file-processing logic — any `.py` files in the `backend/` directory — **no rebuild is needed**. Simply:

1. Stop the running application (`Ctrl + C`)
2. Run `python main.py` again

## 🛠️ Troubleshooting

**`npm run build` fails**
Make sure you are inside the `frontend/` directory when running the command, and that Node.js 20+ is installed (`node --version`).

**`pip install` fails**
Ensure you are using Python 3.11 or higher (`python --version`). Consider using a virtual environment:
```bash
python -m venv venv
source venv/bin/activate   # On Windows: venv\Scripts\activate
pip install -r requirements.txt
```

**The UI doesn't load**
Make sure you have run `npm run build` at least once before starting `python main.py`. The backend serves the compiled frontend files from `frontend/dist/`.
